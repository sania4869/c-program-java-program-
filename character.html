[問題]

次のリスト構造から指定した要素を削除する関数delLLとdelLFをC言語で作成せよ.空欄に当てはまるものを書きなさい.

delLLとdelLFは下記のmain関数において,4つの要素(p1,p2,p3,そして,p4)からなるリスト構造から2個の要素(p4とp1)を,それぞれ,リスト構造の末尾と先頭から削除する.画面表示も示す.空欄はセミコロンを含まない.この問題ではメモリ解放のためのfreeを行う必要はない.

 
          
[画面表示]

Labels: G-L
         
          
          
[解答例]

          空欄1 head->next != NULL
	  空欄2 head->next = NULL
	  空欄3 head->next = head->next->next

            5      char label;
            6      struct dummy_list *next;
            7  } LIST;

           10      if(head->next != NULL) {
           11          printf("%c-", head->label);
           12          dispL(head->next);

           25      }

7〜8行目、
LIST という構造体型を定義しており、各ノードは char 型の label と
次のノードを指す LIST* next をメンバとして持つ。
これにより 1文字のデータと次の要素へのポインタを持つ連結リストを
構築できる。

9〜17行目、
dispL 関数では、引数として受け取った head の指すノードの内容を
再帰的に表示する。まず head->next が NULL でない場合は末尾ではないため、
現在の label を printf で表示し、その後 dispL を head->next を渡して
再帰的に呼び出す。head->next が NULL の場合はそのノードが末尾であるため、
ハイフンなしで label を表示してから return し、再帰呼び出しから
順に戻っていく仕組みになっている。

19〜26行目、
delLL 関数はリストの末尾ノードを削除するための処理を行う。
head->next->next を調べることで、自分の次のノードが末尾かどうかを
判定している。もし head->next->next が NULL でなければ末尾ではないため、
さらに次のノードを引数にして delLL を再帰呼び出しする。
head->next->next が NULL の場合は head->next が末尾要素であると判断し、
そのポインタを NULL に変更することで末尾を切り離して削除している。
実装では free は行っていないが、ここでは末尾を「無効化する」という意味の
削除操作を行っている。

28〜32行目、
delLF 関数は先頭の次のノード、つまりリストの先頭要素を1つ削除する機能を持つ。
head->next が存在する場合に限り、head->next = head->next->next とすることで、
1番目のノードを飛ばして2番目のノードへつなぎ直している。この操作によって、
元の1番目の要素はリストから切り離され、先頭が1つ後ろへ移動する。

34〜41行目、
main 関数ではまず LIST 型の変数 aa0〜aa4 を宣言し、それぞれのノードに
label と next を初期化している。続いて head, p1〜p4 という LIST* 型の
ポインタ変数を宣言し、これらに aa0〜aa4 のアドレスを代入することで、
各ノードを指すポインタを準備している。

44〜48行目、
準備したポインタを利用して連結リストを構築していく。
head->next = p1 によりダミーの先頭 aa0 の次を aa1 にし、
続いて p1->next = p2、p2->next = p3、p3->next = p4 を行うことで、
A→G→L→M の順にノードがつながったリストが完成する。

49行目、
delLL(head) を実行することで末尾削除関数が呼び出され、
リストの最後にある M のノードを切り離し、A→G→L となる。
先頭（ダミーノード）はデータとして数えない。

50行目、
次に delLF(head) を実行することで先頭のデータノードを削除し、
A のノードが外されて G→L の2つのノードからなるリストに変化する。
